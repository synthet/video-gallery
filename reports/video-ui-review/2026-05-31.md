### Failed / partial audits

1. **`color-contrast`** — “Clear filters” danger text on tertiary button background (~**3.82:1** measured in Playwright; needs **≥ 4.5:1** for normal text).
2. **`select-name`** — Capture date and Sort `<select>` elements lack associated `<label>` / `aria-label`.

### Passing highlights

- No console errors logged (Lighthouse + Playwright).
- Active rating pill **4.51:1** (white on `#007acc`) — borderline pass at 11px-equivalent sizing.
- Search input text **7.87:1** on `#333` field.

## Findings (prioritized)

| Area | Severity | Observation | Suggested fix |
|------|----------|-------------|---------------|
| A11y — selects | Medium | Date/sort dropdowns without accessible names | Add `<label htmlFor>` or `aria-label` on `FilterPanel` selects |
| A11y — contrast | Medium | “Clear filters” red on `#2d2d30` ~3.82:1 | Use `var(--color-danger)` on darker bg or outline button style |
| Responsive default | Low | Sidebar open by default at 1024/900 | Optionally default `sidebarOpen=false` under 1100px |
| Modals | Low | Escape key ignored | Add keyboard dismiss |
| Thumbnails / duration | Info | Placeholders + `00:00` in grid | Data pipeline (scan/metadata), not CSS regression |
| Electron CDP | Info | Not run this session | `npm run dev` + MCP `cdp_screenshot` / `cdp_console_logs` for renderer parity |
